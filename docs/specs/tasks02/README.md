# Tasks 02

Este diretorio organiza a implementacao do [PRD 02](C:\Users\jlmac\OneDrive\Documentos\dev\3P\docs\specs\PRD_02\PRD.md) em sprints pequenas, com foco em:

- uma alteracao minima por sprint;
- teste criado ou ajustado antes da implementacao;
- uma mudanca minima no codigo por sprint;
- documentacao objetiva do que foi executado.

## Estrategia

A `task02` foi dividida em duas camadas pequenas:

1. `sprint-01.md`: estrutura inicial das subpaginas de `concepts`.
2. `sprint-02.md`: componente de navegacao reutilizavel entre as subpaginas de conceitos.
3. `sprint-03.md` a `sprint-11.md`: aprofundamento incremental, com uma subpagina por sprint.
4. `sprint-12.md`: revisao textual e gramatical do conteudo de `concepts` em portugues brasileiro.
5. `sprint-13.md`: refinamento pedagogico da continuidade entre as paginas de conceitos.
6. `sprint-14.md`: inclusao de exemplos guiados minimos nas paginas de conceitos.

Essa ordem segue o PRD mestre:

1. `basics`
2. `conditionals`
3. `loops`
4. `decomposition`
5. `lists`
6. `matrices`
7. `strings`
8. `math`
9. `ad-hoc`

## Regra geral

Cada sprint deve:

- revisar os testes existentes antes de alterar o codigo;
- criar ou ajustar os testes primeiro;
- executar os testes e confirmar a falha esperada;
- implementar o menor incremento util;
- executar novamente os testes relevantes;
- registrar a entrega da sprint em linguagem objetiva.
