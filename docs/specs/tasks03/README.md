# Tasks 03

Este diretorio organiza a implementacao do [PRD 03](C:\Users\jlmac\OneDrive\Documentos\dev\3P\docs\specs\PRD_03\PRD.md) em sprints pequenas, com foco em:

- uma alteracao minima por sprint;
- teste criado ou ajustado antes da implementacao;
- uma mudanca minima no codigo por sprint;
- documentacao objetiva do que foi executado.

## Estrategia

A `task03` foi dividida em camadas pequenas para controlar o volume alto de exercicios:

1. `sprint-01.md`: rota e pagina base de `exercises`.
2. `sprint-02.md`: contrato de dados de subcategorias e exercicios.
3. `sprint-03.md`: primeiro container expansivel validando interacao minima.
4. `sprint-04.md`: componente reutilizavel de subcategoria expansivel.
5. `sprint-05.md`: primeiro lote real de exercicios mapeados.
6. `sprint-06.md`: ampliacao do mapeamento para mais subcategorias.
7. `sprint-07.md`: cobertura completa do conjunto principal de exercicios.
8. `sprint-08.md`: revisao textual, consistencia pedagogica e navegabilidade.

## Regra geral

Cada sprint deve:

- revisar os testes existentes antes de alterar o codigo;
- criar ou ajustar os testes primeiro;
- executar os testes e confirmar a falha esperada;
- implementar o menor incremento util;
- executar novamente os testes relevantes;
- registrar a entrega da sprint em linguagem objetiva.

## Regra de linguagem

Todo conteudo visivel da `task03` deve usar portugues brasileiro correto, com acentos, cedilhas e grafia revisada em:

- titulos de subcategorias;
- nomes e descricoes de exercicios;
- textos de abertura, ajuda e navegacao;
- qualquer outro texto renderizado na interface.
