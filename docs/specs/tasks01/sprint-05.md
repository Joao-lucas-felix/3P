# Sprint 05

## Objetivo

Adicionar os conceitos de estruturas de dados introdutorias e manipulacao textual.

## Alteracao minima esperada

- adicionar o bloco de Listas;
- adicionar o bloco de Matrizes;
- adicionar o bloco de Strings.

## Valor entregue

Esta sprint amplia a pagina para temas que exigem mais organizacao de dados e leitura detalhada.

## Teste a criar ou ajustar antes da implementacao

- teste que valida a existencia dos blocos de Listas, Matrizes e Strings;
- teste que valida que cada bloco possui indicacao do tipo de exercicio esperado;
- teste que valida a continuidade visual e estrutural entre os blocos.

## Evidencia de falha esperada antes de implementar

- os blocos ainda nao existem na pagina;
- os testes relacionados a esses conceitos devem falhar antes da implementacao.

## Implementacao esperada

- adicionar tres novos blocos seguindo o modelo existente;
- preservar clareza de leitura;
- manter a pagina escaneavel e organizada.

## Documentacao da execucao

Ao concluir a sprint, registrar:

- quais conceitos foram adicionados;
- como a pagina continua organizada mesmo com aumento de densidade;
- quais partes do layout ou composicao foram reaproveitadas.

## Criterio de conclusao

- os tres blocos estao disponiveis;
- a pagina continua clara para iniciantes;
- os testes da sprint passam.

## Registro de execucao

- testes ampliados para exigir os blocos de Listas, Matrizes e Strings;
- cada novo bloco passou a comunicar tambem o tipo de exercicio esperado;
- os tres conceitos foram adicionados reaproveitando a mesma estrutura de card e a mesma colecao de dados usada nas sprints anteriores;
- a pagina manteve a mesma organizacao visual mesmo com o aumento do numero de blocos.
